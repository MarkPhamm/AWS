# EC2 (Elastic Compute Cloud)

## What is EC2?

EC2 is a virtual server (VM) you rent from AWS. You pick the OS, CPU, RAM, and storage. You only pay while it's running.

### Key Concepts

| Concept | What it is | Example |
| ------- | ---------- | ------- |
| **Instance** | A running virtual server | `i-09c4229a205018058` |
| **AMI** | OS template (like a disk image) | Amazon Linux 2023, Ubuntu |
| **Instance type** | CPU + RAM size | `t3.micro` (2 vCPU, 1 GB) |
| **Key pair** | SSH key for login | `aws-learning-ec2-key` |
| **Security group** | Firewall rules | Allow SSH (22), HTTP (80) |
| **EBS volume** | Disk storage (persists after stop) | 8 GB gp2 SSD (default) |

### Instance Type Naming

```text
t3.micro
│ │  └── size (nano < micro < small < medium < large < xlarge)
│ └── generation (higher = newer, better price/performance)
└── family (t = burstable, m = general, c = compute, r = memory)
```

### Common Instance Types

| Type | vCPU | RAM | Cost | Free tier? |
|------|------|-----|------|------------|
| `t3.micro` | 2 | 1 GB | ~$0.0104/hr | Yes (750 hrs/month) |
| `t3.small` | 2 | 2 GB | ~$0.0208/hr | No |
| `t3.medium` | 2 | 4 GB | ~$0.0416/hr | No |
| `m5.large` | 2 | 8 GB | ~$0.096/hr | No |

Newer accounts get `t3.micro` as free tier. Older accounts get `t2.micro`.

### Instance Lifecycle

```text
Launch → Running → Stop → Stopped → Start → Running → Terminate (deleted)
              │                                             │
              └── Reboot (stays running) ──────────────────┘
```

- **Stop**: Instance pauses. You stop paying for compute, but EBS storage still costs.
- **Terminate**: Instance is deleted. EBS volume is deleted too (by default).
- **Reboot**: Quick restart, keeps the same public IP.

### AMI (Amazon Machine Image)

An AMI is the starting template. It defines OS + pre-installed software.

| AMI | Default user | Package manager |
|-----|-------------|-----------------|
| Amazon Linux 2023 | `ec2-user` | `yum` / `dnf` |
| Ubuntu | `ubuntu` | `apt` |
| Windows Server | `Administrator` | N/A |

You can also create custom AMIs from a running instance (snapshot your server as a template).

### Key Pair

SSH key pair for authentication (instead of passwords).

```text
Your laptop (private key) ---SSH---> EC2 (public key)
```

- Generate locally: `ssh-keygen -t ed25519 -f ~/.ssh/aws-ec2-key`
- Upload public key to AWS via Terraform
- Connect: `ssh -i ~/.ssh/aws-ec2-key ec2-user@<public-ip>`

One key can be assigned to multiple instances. An instance can only have one key pair at launch.

### User Data

A script that runs automatically when an instance first launches. Useful for installing software, configuring the server, etc.

```bash
#!/bin/bash
yum update -y
yum install -y python3 docker git
```

## How We Use EC2

A single `t3.micro` instance in a public subnet for learning and experimentation.

```text
Internet
    ↕
Internet Gateway
    ↕
Public Subnet A (us-east-1a)
    ↕
EC2 Instance (t3.micro, Amazon Linux 2023)
  - SSH on port 22
  - HTTP on port 80
```

## Quick Start

### 1. Generate SSH key

```bash
ssh-keygen -t ed25519 -f ~/.ssh/aws-ec2-key
```

### 2. Deploy with Terraform

Add public key to `terraform.tfvars`:

```hcl
ec2_public_key = "ssh-ed25519 AAAA... your@email"
```

Then:

```bash
terraform apply -target=aws_instance.learning
```

### 3. Connect

```bash
ssh -i ~/.ssh/aws-ec2-key ec2-user@<public-ip>
```

Type `yes` when asked about host authenticity (first-time only).

### 4. Explore the server

```bash
whoami                  # see your username (ec2-user)
uname -a                # see the OS info
free -h                 # check RAM (1 GB)
df -h                   # check disk space (8 GB)
nproc                   # check CPU cores (2)
curl ifconfig.me        # see the public IP from the server's perspective
```

### 5. Install and run stuff

```bash
sudo yum install -y python3    # install Python
python3 --version
sudo yum install -y docker     # install Docker
sudo yum install -y git        # install Git
```

### 6. Run a quick web server

```bash
echo "hello from EC2" > index.html
sudo python3 -m http.server 80
```

Then visit `http://<your-ec2-public-ip>` in your browser. Works because port 80 is open in the security group. Press `Ctrl+C` to stop.

### 7. Clean up

```bash
terraform destroy -target=aws_instance.learning
```

## Terraform Resources

| Resource | Purpose |
|----------|---------|
| `data.aws_ami.amazon_linux` | Look up latest Amazon Linux 2023 AMI |
| `aws_key_pair.ec2` | Upload SSH public key to AWS |
| `aws_security_group.ec2` | Firewall: SSH + HTTP in, all out |
| `aws_instance.learning` | The EC2 instance itself |

All defined in `aws_ec2.tf`.

## Cost

- **t3.micro**: Free tier (750 hrs/month for 12 months)
- **EBS storage**: ~$0.10/GB/month (8 GB default = ~$0.80/month)
- **Public IP**: Free while instance is running
- **Data transfer**: First 100 GB/month outbound is free
- **For this project**: Free within free tier limits. Run `terraform destroy` when done.

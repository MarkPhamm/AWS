# S3 (Simple Storage Service)

## What is S3?

S3 is a cloud file storage service. Think of it as a Google Drive / Dropbox
that your code and AWS services can read from and write to.

### Key Concepts

| Concept | Analogy | Example |
| ------- | ------- | ------- |
| **Bucket** | A USB drive or top-level folder | `airflow-mwaa-203110101827` |
| **Object** | A file inside that bucket | `dags/my_dag.py` |
| **Key** | The file path (relative to bucket root) | `dags/my_dag.py` |
| **Versioning** | Git history but for files | Each upload creates a new VersionId |

### Bucket Naming Rules

- Must be **globally unique** across ALL AWS accounts worldwide
- 3-63 characters, lowercase letters, numbers, and hyphens only
- Cannot start or end with a hyphen
- Convention: include your account ID or org name to guarantee uniqueness

### Versioning

When versioning is enabled, S3 keeps every version of a file, not just the latest.

Without versioning:

- Upload `requirements.txt` → v1 exists
- Upload `requirements.txt` again → v1 is gone, only v2 exists

With versioning:

- Upload `requirements.txt` → v1 exists (VersionId: abc123)
- Upload `requirements.txt` again → v1 AND v2 exist (VersionId: def456)

MWAA needs versioning because it references a specific version of `requirements.txt`:

```bash
aws s3api head-object --bucket my-bucket --key requirements.txt --query 'VersionId'
```

### Public Access Block

By default, S3 buckets are private. The public access block is extra protection
that prevents anyone from accidentally making the bucket or its files public.
Always enable this unless you specifically need public access (e.g., hosting a website).

## How We Use S3 for MWAA

```text
airflow-mwaa-203110101827/      ← the bucket
├── dags/                        ← DAG files (MWAA auto-reads every 30s)
│   ├── my_dag.py
│   └── dbt/
└── requirements.txt             ← Python packages (MWAA installs on update)
```

The CI/CD pipeline (deploy.yml) syncs files to this bucket:

```yaml
aws s3 sync dags s3://airflow-mwaa-203110101827/dags --delete
aws s3 cp requirements.txt s3://airflow-mwaa-203110101827/requirements.txt
```

## Terraform Resources

Creating an S3 bucket in Terraform requires 3 separate resources:

1. **`aws_s3_bucket`** - Creates the bucket itself
2. **`aws_s3_bucket_versioning`** - Enables versioning (separate setting)
3. **`aws_s3_bucket_public_access_block`** - Blocks public access (separate setting)

Why 3 resources instead of 1? AWS treats bucket settings as independent from the bucket.
Terraform mirrors this. Each resource links back to the bucket with `bucket = aws_s3_bucket.mwaa.id`.

## Cost

- **Storage**: $0.023/GB per month (first 50TB). Your DAGs are tiny, so essentially free.
- **Requests**: $0.005 per 1,000 PUT requests, $0.0004 per 1,000 GET requests.
- **Free tier**: 5GB storage, 20,000 GET, 2,000 PUT requests per month.
- **For this project**: Practically $0. DAGs are small text files.
